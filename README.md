# 活动住宿需求统计系统

这是一个用于活动期间统计住宿需求、酒店安排和房型数量的样机。前端是静态页面，生产部署时通过 `/api/state` 接口把数据同步到飞书多维表格。

## 如何打开

双击或用浏览器打开：

`index.html`

如果直接打开本地文件，系统会使用浏览器本地数据。部署到支持 Node/Vercel Serverless API 的环境后，系统会优先连接飞书多维表格共享数据；页面右上角会显示同步状态。

仍然可以用页面右上角的“导出 JSON”和“导入 JSON”备份或恢复数据。

## 多人同步

同步链路：

`网页 -> /api/state -> 飞书 OpenAPI -> 飞书多维表格`

飞书多维表格：

- Base：`https://ucngneehl818.feishu.cn/base/Mg3abeaEya2QptsxOjIchxSLndd`
- 快照备份表：`系统同步数据`
- 快照备份表 ID：`tblFP1unaKOU0CCR`
- 记录键：`hotel-room-state-v1`
- 快照备份表链接：`https://ucngneehl818.feishu.cn/base/Mg3abeaEya2QptsxOjIchxSLndd?table=tblFP1unaKOU0CCR`

方案 B 之后，网站实际读写以下核心表：

- `住宿需求核心表`：一条住宿需求一行，保存日期、酒店、房间号、房型和备注。
- `住宿人员明细表`：一个人一行，通过需求ID归属到某条住宿需求。
- `操作记录表`：记录网站新增、编辑、删除、批量导入和旧数据迁移。

为了方便在飞书里直接查看，系统会继续自动维护几张展示表：

- `住宿人员名单`：一人一行，方便按姓名、人员性质、酒店、房间号快速筛选。
- `酒店统计查看`：一行一个“日期 + 酒店 + 人员性质”的房型统计，方便按酒店查看每天各角色占房情况。
- `角色统计查看`：一行一个“日期 + 人员性质 + 酒店”的房型统计，方便按人员性质查看每天分布在哪些酒店。

这些查看表是从核心表自动整理出来的镜像，适合查看、筛选和排序；正式修改建议回到网站操作。`系统同步数据` 只保留为兼容和备份快照。旧版的 `入住需求查看`、`酒店房间查看`、`分房记录查看`、`变更记录查看` 不再维护，会在同步时尝试清理。

后端环境变量参考 `.env.example`：

- `FEISHU_APP_ID`
- `FEISHU_APP_SECRET`
- `FEISHU_BASE_URL`
- `FEISHU_APP_TOKEN`
- `FEISHU_SYNC_TABLE_ID`
- `FEISHU_SYNC_TABLE_NAME`
- `FEISHU_SYNC_RECORD_KEY`

注意：`FEISHU_APP_SECRET` 不能写进 `app.js`、`index.html` 或 GitHub 公开仓库，只能配置在后端部署平台的环境变量里。

当前已给默认协作者配置 `full_access` 管理权限。

## 当前业务口径

- 核心数据只有“入住需求”：一条需求可以包含一个人或多个人。
- 每条需求维护人员信息、入住/离店日期、安排酒店、房间号、房间类型和备注。
- 酒店统计按“日期 + 酒店 + 房型”统计每天安排了多少间。
- 角色统计按“日期 + 人员性质 + 房型”查看不同角色的住宿分布。
- 系统不再维护独立的酒店房间库存、分房记录、现场核对和变更记录。

## 主要视图

左侧菜单栏保留四个核心模块：

- 总览：核心指标、待补信息、人员性质分布和酒店日期统计。
- 入住需求：维护人员信息、日期、安排酒店、房间号、房间类型和备注。
- 酒店统计：按酒店和日期查看各房型安排数量。
- 角色统计：按人员性质和日期查看各房型安排数量。
