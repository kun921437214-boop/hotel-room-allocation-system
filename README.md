# 本地酒店房间分配系统

这是一个参考 MRBS（Meeting Room Booking System）思路改造的活动酒店房间分配样机。前端是静态页面，生产部署时通过 `/api/state` 接口把数据同步到飞书多维表格。

## 如何打开

双击或用浏览器打开：

`index.html`

如果直接打开本地文件，系统会使用浏览器本地数据。部署到支持 Node/Vercel Serverless API 的环境后，系统会优先连接飞书多维表格共享数据；页面右上角会显示同步状态。

仍然可以用页面右上角的“导出 JSON”和“导入 JSON”备份或恢复数据。

## 参考 MRBS 的地方

- MRBS 的 `Area` 思路：本系统里对应“酒店”。
- MRBS 的 `Room` 思路：本系统里对应“房间”。
- MRBS 的 `Entry` 思路：本系统里对应“一次入住分配”。
- MRBS 的冲突检测：同一房间在重叠日期内不能重复分配。
- MRBS 的日历视图：本系统改成“房晚日历”，按房间行和日期列展示状态。

## 为活动酒店场景改造的地方

- 加入入住需求表，而不是只记录会议预订。
- 加入身份类型、人数、儿童数、期望房型、特殊需求。
- 在样例数据里保留到店状态和变更记录，方便总览看待处理事项。
- 首页改成运营看板，优先显示剩余房晚、未分配需求、未确认分配和待处理变更。

## 主要视图

左侧菜单栏只保留前五个核心模块：

- 总览：核心指标、酒店日期房量热力表、今日待处理、用途分布。
- 房晚日历：按酒店、日期查看每间房每天的占用状态。
- 分房：选择入住需求和房间，自动检查日期冲突。
- 入住需求：维护个人、家庭或团队的住宿需求。
- 酒店房间：维护酒店、房间号、房型、楼层、可住人数、可用时间范围、默认用途等。

## 当前数据状态

- 已预置酒店：汉庭酒店、如家酒店、万豪酒店。
- 房间、入住需求、分房记录、变更记录均为空。
- 页面会保留功能结构和空状态提示，方便从零开始录入。
- 新增房间时需要在同一个“可用时间范围”里同时设置开始和结束；默认是 2026-08-01 至 2026-08-06，房晚日历和分房可选房间会按这个范围计算。
